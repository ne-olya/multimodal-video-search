# VideoScope: мультимодальный поиск по видео

VideoScope индексирует реальное содержимое роликов и ищет их по свободному тексту. Для каждого видео система извлекает ключевые кадры, речь, текст на экране и метаданные, строит эмбеддинги и возвращает ранжированный результат с доказательствами совпадения.

## Возможности

- равномерный и scene-aware выбор кадров;
- OpenCLIP для общей области изображений и запросов;
- Faster Whisper для ASR и EasyOCR для надписей в кадре;
- общая CLIP text tower для речевого и OCR-текста, совместимая по размерности с visual embedding;
- late fusion с перенормировкой весов при отсутствии модальности;
- FAISS `IndexFlatIP`, сохраняемый как `index.faiss`, плюс NumPy fallback и JSON manifest;
- Recall@K, Precision@K, MRR и интерактивное Streamlit-демо.
- segment-level retrieval по временным окнам, агрегация лучшего фрагмента на видео;
- раздельные visual/ASR/OCR scores и численное объяснение вклада каждой модальности.

## Установка и запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e '.[ml,demo,dev]'
mkdir -p data/videos
# поместите видео в data/videos
videoscope-index --config configs/default.yaml
streamlit run app.py
```

При первом запуске модели загружают веса. Базовый индекс использует точный `faiss.IndexFlatIP`; для большой коллекции замените его на HNSW/IVF. Все векторы L2-нормированы, поэтому inner product равен cosine similarity. Если FAISS недоступен, автоматически используется NumPy fallback.

## Pipeline

1. `sampling.py` выбирает временные позиции и декодирует RGB-кадры.
2. Кадры группируются в временные сегменты; короткое событие не растворяется в среднем по ролику.
3. Faster Whisper формирует transcript и сегменты с timestamps.
4. EasyOCR читает текст на выбранных кадрах и удаляет точные дубли.
5. Текстовые модальности кодируются text tower той же CLIP-модели и объединяются с visual embedding.
6. `manifest.json` хранит пути, ключевые кадры и тексты; `vectors.npy` — нормированные векторы.

Для русскоязычной коллекции качество text tower нужно проверить отдельно. Улучшенный вариант использует multilingual Sentence Transformer и обучаемые projection heads, приводящие visual/text в общую размерность. Для быстрого visual-only эксперимента отключите ASR/OCR.

## Эксперименты

Используйте один test-набор запросов с ручной множественной релевантностью: один запрос может соответствовать нескольким роликам. Рекомендуемый минимум — 100 запросов разных типов: объект, действие, речь, надпись и их сочетание.

| Вариант | Visual | ASR | OCR | Что проверяет |
|---|---:|---:|---:|---|
| video | да | нет | нет | базовое визуальное содержание |
| video + speech | да | да | нет | пользу речи и диалогов |
| all | да | да | да | пользу надписей, титров и интерфейсов |

Отдельно сравните uniform 4/8/12/24 кадров, scene-aware sampling и ограничение по одному кадру на сцену. Измеряйте Recall@1/5/10, MRR, Precision@5, p50/p95 времени поиска, время индексации на минуту видео и размер индекса. Для статистической честности используйте bootstrap confidence interval по запросам.

Формат оценочного JSONL:

```json
{"query":"девушка делает макияж перед зеркалом","relevant_ids":["video_17"],"retrieved_ids":["video_17","video_03"]}
```

```bash
videoscope-evaluate reports/retrieval.jsonl --k 10
videoscope-benchmark reports/relevance.jsonl --index artifacts/default
python scripts/plot_results.py --input reports/results.csv
pytest -q
```

## Анализ ошибок

Разделяйте ошибки на пропущенное короткое событие, визуально похожее действие, неправильный ASR, OCR-шум, языковой mismatch и неверный fusion-вес. В demo объяснение перечисляет реально доступные источники сигнала; для более строгого объяснения сохраняйте вклад cosine score каждой модальности и наиболее близкий кадр.

## Ограничения и production

Репозиторий не поставляет видео и веса. Перед публикацией следует проверять лицензии, согласие на обработку речи и лиц, удалять персональные данные из transcript и ограничивать доступ к исходным роликам. Для production нужны очередь фоновой индексации, объектное хранилище, FAISS/векторная БД, дедупликация, контроль версий encoder, фильтры доступа и offline/online мониторинг качества.
